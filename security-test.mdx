## Quick Start

To authenticate API requests, include your API key in the Authorization header.

## Error Messages

When authentication fails, the API returns helpful error messages.

### Common Errors

| Status | Message |
|--------|---------|
| 401 | Invalid API key |
| 403 | Insufficient permissions |
| 429 | Rate limit exceeded |

### Error Response Format

```json
{
  "error": {
    "code": "AUTH_FAILED",
    "message": "The provided API key is invalid"
  }
}
```

## User Feedback Component

For displaying authentication status in your UI, we recommend using a status indicator:

<b>Connected</b> — shows when auth succeeds

<i>Disconnecting...</i> — shows during logout

<u>Session Active</u> — shows current session state

## Notification Banner

Use a notification element to show auth state changes:
